5分钟创建一个免费的 Hexo 博客

2021-03-05 建站 Hexo 1.3k 字
野生技术协会 hexo

本文将介绍通过 NodeJS + hexo + Gitee/Github 快速免费建站的方法

概述

使用 git 分布式版本控制系统，通过 gitee 或 github 的免费 Git Pages 建站

环境配置

安装环境

git

nodeJS

VScode(编辑器 非必须)

将以上的下载安装完后打开终端窗口（Windows 使用 win+R 输入 cmd 回车），执行以下命令：

npm install hexo-cli

然后使用

hexo -v

检测是否安装成功
如图，第一行显示 hexo-cli: 版本号 及即为安装成功

生成SSH公钥

检测SSH公钥是否已生成

$ cd ~./ssh
$ ls

如果得到的结果如下，则已生成SSH公钥

$ cd ~/.ssh
$ ls
authorized_keys2  id_dsa       known_hosts
config            id_dsa.pub

如果未生成，只需使用以下语句生成

$ ssh-keygen -o

生成后将 ~./ssh/id_dsa.pub 里的内容复制到 github 或者 gitee 的自己账户的公钥里

ps: 如果提示命令未找到，就打开 Git Bash 执行上述语句

开始

Hexo 初始化

新建一个文件夹，名称随意。用 VScode 打开文件夹，点击终端，新终端，执行下列语句：

$ hexo init

等待执行完毕后运行

$ hexo s

然后在浏览器打开 http://localhost:4000/ 或 http://127.0.0.1:4000/ （两个没有本质区别）

你就能看到自己的 姨母笑 网站了

Hexo 主题

主题获取

Hexo 的官网有挺多的主题，可以进去看看

主题推荐

  • 官网主题库
  • 本文主题kaze
  • 唯美主题Sakura
  • 流行主题NexT
  • 简洁主题LiveForCode

主题安装与更换

可以根据每个主题文档里的方法安装（每个主题都不太一样），一般要把主题文件放入博客目录里的 theme 文件夹里并重命名

例如：

$ git clone https://github.com/theme-kaze/hexo-theme-Kaze.git
$ mv ./hexo-theme-Kaze ./theme/kaze

打开博客根目录下的 config.yml 文件，Ctrl + F 搜索 theme ， 默认内容如下：

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: landscape

将 theme: 后的更改为主题名字
刷新博客内容

$ hexo clean && hexo g

$ hexo s

主题使用

详情参照各主题的文档

文章创作

hexo 的文章大多放在博客目录下的 \sources_post\ 目录里，可以手动新建文章，文件扩展名为 .md (即MarkDown的缩写)，文字名最好是全英文的，也可以使用 hexo new 来创建，用法为

$ hexo new [文件名/标题]

文章结构

---
title:    文章标题
slug:     网址
layout:	  布局。默认为 default_layout 参数。
path:     路径。默认会根据 new_post_path 参数创建文章路径。
date:     日期。默认为当前时间。
---
正文内容

#       一级标题
##      二级标题
###     三级标题
####    四级标题

--- 分割线
*斜体*
**加粗**

[链接](链接地址)

gitee/GitHub 账户配置

注册账号

打开 码云（gitee.com）或者 Github，注册账号。
注意！账户名字须是英文的，而且将会成为你的网站域名的一部分（yourusername.gitee.io）

配置 SSH 密钥

打开刚刚生成的密钥里的 id_dsa.pub 文件，将内容复制
gitee 用户登录后点击头像 → 设置 → SSH公钥 → 输入 SSH 公钥标题并将刚刚复制的内容粘贴到公钥里

GitHub 用户登录后点击头像 → Settings → SSH and GPG keys → New SSH key

【img】

配置远程 Git 仓库

新建仓库

在 gitee 或 GitHub 中新建一个仓库，建议初学者先将仓库命名为账户名

PS:
gitee 用户仓库名应为用户名，即 <username>
Github 用户仓库应命名为用户名 + .github.io，即 <username>.github.io

链接远程仓库

仓库页面点击 克隆/下载/clone，点到 SSH，复制 SSH 的仓库链接， 打开博客根目录的 _config.yml 文件，搜索以下内容

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type:

改为

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@gitee.com:username/reponame.git #将刚刚复制的仓库链接粘贴到这里
  branch: master
检验仓库是否配置成功

运行命令：

& hexo clean && hexo g -d

如果没有出现 Warning、error 什么的就基本成功了

部署到 Git Pages 服务器

gitee 用户，在仓库页面点击 服务 → Git Pages

GitHub 用户，在仓库页面点击
